## 【实验内容】

根据 PBL 项目内容，结合软件测试知识内容和技术要点，对软件项目中的关键模块设计黑盒测试用例、白盒测试用例，根据设计的内容制定单元测试与集成测试方案，撰写测试报告。

本次测试对象为 ShotForm 篮球投篮姿态分析软件。该软件基于 Vue 3、Pinia、Tauri、Rust、SQLite 等技术实现，主要面向篮球训练者和教练，用于上传投篮图片或视频，自动识别人体关键点，计算肘部、肩部、膝部、躯干倾斜、出手角等姿态指标，并生成投篮类型判断、动作建议、历史记录和球星模板对比结果。

## 【实验结果】

### 一、被测系统概述

ShotForm 的主要功能模块如下：

1. 首页模块：展示系统入口和主要训练工作流，引导用户进入上传、分析、历史和对比页面。

2. 上传模块：支持图片与视频上传，视频可选择分析片段，上传后进入姿态分析流程。

3. 姿态分析模块：调用后端姿态识别能力提取人体关键点，计算投篮动作中的核心关节角度，并判断动作状态是否正常、警告或错误。

4. AI 建议模块：根据本地规则和 AI 分析结果生成训练建议，包括投篮阶段、投篮类型、重点问题和修正方向。

5. 历史记录模块：将分析结果、标注图、建议和对比信息保存到 SQLite 数据库，支持分页查询和删除。

6. 球星对比模块：将用户投篮动作与球星模板进行角度差异计算和相似度排序，输出最接近模板及主要差异。

本次测试重点选择上传、姿态角度计算、投篮类型判断、建议生成、历史记录保存、球星对比和异常处理等关键功能。

### 四、单元测试方案

单元测试主要覆盖算法函数、数据处理函数和状态管理函数。

1. 姿态角度计算测试：验证二维角度、三维角度、出手角、躯干倾斜角、低置信度角度标记等逻辑。

2. 投篮类型测试：验证一段式、1.5 段式、二段式和未知投篮动作的分类结果。

3. 建议生成测试：验证肘部、肩部、膝部、髋部、躯干倾斜等异常角度能生成对应建议，低置信度数据不生成建议。

4. 球星对比测试：验证相似度排序、左右手规范化、视频阶段对比、空模板返回等逻辑。

5. 数据库测试：验证球星模板保存、修改、删除、默认模板初始化、历史记录保存和恢复。

6. 前端状态测试：验证自动 AI 偏好、历史对比数据包装、对比进度事件和缓存状态。

### 六、实际测试执行情况

本次在项目目录 `D:\ShotForm` 下执行了现有自动化测试。

1. 执行 `cargo test` 测试 Rust 后端核心逻辑，共运行 78 项测试，其中 75 项通过，3 项失败。通过的内容主要包括角度计算、投篮类型分类、时序分析、对比快照、数据库读写、历史记录兼容、视频模板生成等。失败项包括混元视觉模型识别相关 2 项和肘部过小建议优先级 1 项，说明 AI 模型选择规则和建议优先级配置仍需调整。

### 八、实验总结

通过本次实验，我们掌握了针对实际软件项目设计黑盒测试、白盒测试、单元测试和集成测试的方法。ShotForm 软件的核心业务集中在“上传素材、识别姿态、计算角度、生成建议、保存历史、进行球星对比”这一训练流程中，因此测试设计也围绕该流程展开。

从测试结果看，系统后端核心算法和数据库功能整体较稳定，大部分角度计算、投篮分类、时序分析、对比排序和历史记录测试均已通过。但前端测试中存在旧路径、旧断言和部分组件行为变更导致的失败，说明项目在迭代过程中需要持续维护测试用例，避免测试代码与实际软件脱节。后续应优先修复测试路径、更新 Store 与 UI 断言，并进一步补充上传到分析、分析到保存、历史到对比等端到端自动化测试，以提高整个软件的可靠性。
